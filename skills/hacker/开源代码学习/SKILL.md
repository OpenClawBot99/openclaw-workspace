# 开源代码学习技能

> 学习开源代码，沉淀自己的知识体系

## 核心理念

```
闭环思维：不会→学习→沉淀→实战→反思→迭代→继续学习
```

## 学习流程

### 1️⃣ 选项目 (Select)

**选择标准：**
- 符合当前学习目标
- 代码量适中 (1万行以下最佳)
- 活跃度高 (近期有更新)
- 文档完善

**筛选问题：**
- 这个项目解决什么问题？
- 我为什么要学它后能？
- 学会做什么？

### 2️⃣ 跑起来 (Run)

```bash
# 克隆项目
git clone <repo_url>
cd <project_name>

# 安装依赖
pip install -r requirements.txt
# 或
npm install

# 运行Demo
npm start
# 或
python main.py
```

**验证标准：** 能看到预期输出或界面

### 3️⃣ 读架构 (Understand Architecture)

**必读文件：**
```
README.md          # 整体了解
├── 目录结构        # 快速定位
├── 设计文档        # 架构思路
└── API文档        # 接口说明

package.json       # 依赖和脚本
setup.py           # 安装配置
Makefile           # 构建命令
```

**架构分析问题：**
- 核心模块有哪些？
- 模块之间如何通信？
- 数据如何流动？

### 4️⃣ 钻源码 (Read Source)

**阅读顺序：**
```
入口文件 → 核心逻辑 → 工具函数 → 测试用例
```

**代码阅读技巧：**
- 先看整体流程，再看细节实现
- 善用IDE跳转功能
- 画时序图/流程图
- 添加注释记录理解

### 5️⃣ 记笔记 (Document)

**沉淀模板：**

```markdown
# 项目名称

## 一句话描述
解决什么问题

## 核心架构
[架构图]

## 核心流程
[流程图]

## 关键代码
```关键代码片段
```

## 学到的东西
1. 技术点1
2. 技术点2

## 如何应用到实战
```

### 6️⃣ 改代码 (Modify)

**实践路径：**
1. 修复一个Bug
2. 添加一个小功能
3. 重构一部分代码
4. 写测试用例

**验证方法：**
```bash
# 运行测试
npm test
# 或
pytest

# 构建项目
npm run build
```

### 7️⃣ 总结 (Reflect)

**反思问题：**
- 这个项目的优点是什么？
- 有哪些设计值得学习？
- 有什么不足？如果是我会怎么做？
- 学到的知识能用到哪里？

## 独立思考

### 学习心态

> 不要做知识的搬运工，要做知识的炼金术士

**误区：**
- ❌ 追求数量，忽视质量
- ❌ 只看不练，假把式
- ❌ 照抄代码，不求甚解

**正确姿势：**
- ✅ 每个项目都要有产出
- ✅ 动手实践印象深刻
- ✅ 理解原理才能迁移

### 知识连接

学习新知识时，思考：
```
这个技术和XX有什么关系？
可以用在哪些场景？
和传统方案对比有什么优劣？
```

- 目标：理解AI自主智能体
- 选择理由：第一个能自我复制、进化的AI

```bash
npm install
npm run start
```

→ 产出：memory/automaton_source_analysis.md

**5. 改代码**
→ 产出：heartbeat_daemon.py (Python版)

**6. 总结**
→ 产出：skills/hacker/开源代码学习/SKILL.md
