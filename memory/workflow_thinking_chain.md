# 工作流思维链 - 问题驱动技能增强

## 核心理念
遇到问题 → 问题导向 → 问题驱动 → 增加技能 → 闭环探索

---

## CDP模式测试实战案例

### 1. 遇到问题 (Problem Encountered)
- **问题1**: Chrome CDP启动失败 - 需要用户手动连接扩展
- **问题2**: CDP服务连接不稳定
- **问题3**: 不同profile行为不同

### 2. 问题导向 (Problem-Oriented)
| 问题 | 分析 | 方案 |
|------|------|------|
| 扩展未连接 | chrome profile依赖用户交互 | 尝试openclaw profile |
| 服务连接失败 | CDP服务可能未启动 | 检查服务状态 |
| 操作失败 | profile配置差异 | 记录profile特性 |

### 3. 问题驱动 (Problem-Driven)
- **驱动1**: 用户提示"用skill实现"
- **驱动2**: 建议"测试例子选CDP"
- **驱动3**: 要求"闭环探索思维"

### 4. 增加技能 (Skill Enhancement)
```
Skill更新:
├── automation_control
│   ├── 加入OpenClaw browser工具
│   ├── 加入profile选择策略
│   └── 加入CDP问题排查
└── 认知提升
    ├── Profile机制理解
    ├── CDP工作原理
    └── 浏览器自动化边界
```

### 5. 闭环探索 (Closed-Loop)
```
P(Plan) → D(Do) → C(Check) → A(Act)
   ↑                              │
   └──────────────────────────────┘
```

## 关键原则

### 1. 问题不过夜
- 当天问题当天分析
- 记录在memory中

### 2. 技能必须更新
- 遇到问题→更新Skill
- 避免重复踩坑

### 3. 闭环思维
- Plan → Do → Check → Act
- 持续迭代优化

### 4. 知识沉淀
- 写到Skill中
- 更新memory
- 形成肌肉记忆
