# SSRF(Server-Side Request Forgery)学习笔记

## 什么是SSRF?
SSRF是一种Web安全漏洞，允许攻击者迫使服务器向内部系统或外部服务发起请求。

## 攻击原理

### 1. 基本原理
```
攻击者 → 服务器 → 内部资源
```

### 2. 常见利用点
- URL参数
- 表单提交
- 文件预览

## 攻击场景

### 1. 内部服务访问
- `http://169.254.169.254/` (AWS元数据)
- `http://localhost/admin`
- `http://127.0.0.1:port`

### 2. 端口扫描
- 探测内网开放端口
- 服务识别

### 3. 读取本地文件
- `file:///etc/passwd`
- `gopher://` 协议

## 防御措施

### 1. 限制请求来源
- 验证用户输入URL
- 白名单机制

### 2. 禁用危险协议
- 禁用file://, gopher://, ftp://

### 3. 统一错误信息
- 防止信息泄露

### 4. 定期安全测试

## 实践工具

- Burp Suite
- SSRFmap
- Gopherus
