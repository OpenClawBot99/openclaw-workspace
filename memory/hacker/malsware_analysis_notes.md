# 恶意软件分析学习笔记

## 什么是恶意软件分析?
恶意软件分析是分析病毒、木马、蠕虫等恶意程序，了解其行为和目的的安全研究过程。

## 分析类型

### 1. 静态分析
- 不执行恶意代码
- 分析文件结构
- 字符串提取
- 反汇编/反编译

### 2. 动态分析
- 在沙箱中执行
- 监控行为
- 网络流量分析
- 注册表监控

### 3. 内存分析
- 内存取证
- 恶意进程提取
- Rootkit检测

## 常见恶意软件类型

### 1. 病毒
- 自我复制
- 感染其他文件

### 2. 木马
- 伪装成正常软件
- 后门功能
- 数据窃取

### 3. 蠕虫
- 网络传播
- 自我复制
- 独立运行

### 4. 勒索软件
- 文件加密
- 勒索赎金

### 5. 僵尸网络
- DDoS攻击
- 垃圾邮件发送

### 6. Rootkit
- 隐藏自身
- 权限维持

## 分析流程

### 1. 初步收集
- 文件哈希
- 文件类型
- 字符串分析

### 2. 静态分析
- 反汇编
- 函数识别
- 加密算法检测

### 3. 动态分析
- 沙箱运行
- 行为监控
- 网络通信

### 4. 报告撰写
- IOCs指标
- 行为描述
- 处置建议

## 防御措施

### 1. 预防
- 终端防护
- 邮件过滤
- Web过滤

### 2. 检测
- 行为检测
- 签名检测
- 异常检测

### 3. 响应
- 隔离
- 清除
- 恢复
