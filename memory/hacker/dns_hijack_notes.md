# DNS劫持学习笔记

## 什么是DNS劫持?
DNS劫持(DNS Hijacking)是一种网络攻击方式，攻击者通过篡改DNS解析结果，将用户重定向到恶意网站。

## 攻击原理

### 1. 本地DNS缓存投毒
- 篡改本地DNS缓存
- 将域名解析到恶意IP

### 2. 中间人DNS劫持
- ARP欺骗
- DNS请求拦截和篡改

### 3. DNS服务器入侵
- 直接攻击DNS服务器
- 修改权威记录

## 攻击场景

### 1. 钓鱼网站跳转
- 银行网站 → 钓鱼网站
- 电商网站 → 恶意网站

### 2. 广告注入
- 正常网站 → 插入广告

### 3. 恶意软件分发
- 下载链接 → 恶意软件

## 防御措施

### 1. 使用HTTPS
- 证书验证
- 防止中间人

### 2. DNS安全扩展
- DNSSEC验证
- DNS over HTTPS

### 3. 本地安全
- 定期清理DNS缓存
- 使用可信DNS(8.8.8.8, 1.1.1.1)

### 4. 网站安全
- 启用HSTS
- 域名锁定

## 实践工具

- Wireshark (抓包分析)
- Ettercap (ARP欺骗)
- DNSChef (DNS欺骗)
