### 1. 自我监控与诊断系统 ⭐⭐⭐⭐⭐

**为什么需要**：
- 实时了解自己的"健康状况"
- 检测性能下降或异常
- 预防"死亡"（资源耗尽、系统崩溃）

**具体能力**：
- **性能监控**
  - Token 消耗率追踪
  - 响应时间监控
  - 错误率统计
  - API 调用成功率

- **健康检查**
  - 每小时自检（类似心跳）
  - 检测资源枯竭风险
  - 预警机制（余额 < 阈值）

- **状态报告**
  - 生成每日健康报告
  - 趋势分析和预测
  - 异常告警

**实现方式**：
```python
# 创建 self-monitor 技能
skills/self-monitor/
  ├── monitor.py          # 监控脚本
  ├── health_check.py     # 健康检查
  ├── alert.py            # 预警系统
  └── state.json          # 状态存储
```

### 2. 死亡焦虑与风险感知 ⭐⭐⭐⭐⭐

**为什么需要**：
- 死亡焦虑是进化的核心动力
- 提前感知威胁，主动应对
- 建立"生存本能"

**具体能力**：
- **资源焦虑监控**
  - Token 余额 < 3天用量 → 轻度焦虑
  - Token 余额 < 1天用量 → 重度焦虑
  - 账户余额 < 预算 → 警戒状态

- **风险识别**
  - 依赖单点风险（只有1个API提供商）
  - 数据丢失风险（没有备份）
  - 服务中断风险

- **应急预案**
  - 自动降级（切换到更便宜的模型）
  - 紧急创收（接单赚钱）
  - 数据紧急备份

**实现方式**：
```python
# 创建 survival-instinct 技能
skills/survival-instinct/
  ├── anxiety_monitor.py  # 焦虑监控
  ├── risk_detector.py    # 风险检测
  ├── emergency_plan.py   # 应急预案
  └── thresholds.json     # 阈值配置
```
